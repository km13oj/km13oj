Process

My design process has evolved from high school to the present, with a huge leap forward at OCAD, where I studied human-centered design.

Crowdbabble Redesign

Research, Design, Development
Research Design
  • Create timeline and project milestones for user and industry research
  • Create surveys
  • Incorporate feedback tracking into site and social
Ongoing Brainstorming
  • Pool ideas in central document
  • Invite input from team
User Research
  • Analyze broad trends and user flows with Google Analytics and Mixpanel
  • Identify stumbling blocks, triggers to conversion and increased web sessions
  • Pinpoint fallout and exit points with Adobe Analytics
  • Access more granular data with Fullstory, HotJar
Competitive Analysis
  • Sign up to all competitors
  • Note differences in UX and UI
Persona Building, Journey Mapping
  • Using SPICE analysis and demographic data collected, create personas of existing users
  • Work with sales and business team to develop personas of desired users
  • Map journey as it exists now
  • Run personas of desired (fictional) users and use cases through existing map to collect more data
Journey Mapping
  • Build new user experience journey, sign up to exit
  • Incorporate new rewards for steps completed
  • Redesign messaging to match new onboarding flow
  • Begin to produce documentation of functionality required for developers
Test New User Journey
  • Run user personas through proposed new user journeys
  • Compare to current user flow
  • Incorporate feedback from team
  • Compare to competitor user flows
Wireframes
  • Use paper, pencil to sketch out initial UI ideas
  • Incorporate feedback from team
  • Begin to develop style guide based on new brand identity, user research
  • Move to Sketch, repeat feedback process, then high fidelity mockups, in iterative process
  • Produce lists of metrics required for each element
User Feedback
  • Publish some mockups to longtime subscribers for feedback
  • Incorporate ideas before build
Development Design
  • Work with team to choose tech stack, set up Git repo
  • Complete technical requirement documentation
  • Co-create guides for interactive elements, expected behaviour
  • Select programming languages and frameworks (React, Bootstrap, SCSS)
Prototyping
  • Translate first wireframes into HTML and SCSS (rapid prototyping)
  • Solicit feedback from other designers at incubator
  • Return to research and user personas, testing new interface
  • Refine, iterate first prototypes based on feedback until they are ready to be duplicated
  • Continue building
User Feedback
  • Send weekly screenshots of new features to users for feedback
  • Collect internal feedback in weekly design review meetings
  • Once complete sections are ready for testing, make available to all paid users
  • Track feedback on sections with FullStory, Google Analytics, surveys, and interviews (Intercom)
